Syllabus

Memory Diagrams

  • Draw a memory diagram for scalar, reference, and compound types.
  • Given a memory diagram, write the Clojure code that will implement it.

Records

  • Be able to define a record type in Clojure.
  • Given a record type, write the Clojure code that can instantiate an object of that type.
  • Use selectors to retrieve data from a record.
  • Convert between memory diagrams and Clojure code.
  • You will not have to write deftype or defmtype declarations yourself, but you should be able to write code that uses them.

Sequences

  • Explain the concept of a sequence.
  • Explain how sequences are different than lists.
  • Explain what happens when you call next or rest on a vector.

Vectors

  • Show how to use vectors in Clojure.
  • Explain the trade-offs between vectors and linked storage.
  • Note: Clojure vectors are resizable, but many languaegs do not support that. What are the consequences?
  • Be able to write code such as binary search, running sum, minimum element, etc.

Singly Linked Lists

  • Give the time complexities for sorted-insert, insert-first, insert-last.
  • Give the time complexity for find.
  • Give the time complexity for delete-first, delete, and delete-last.
  • Explain what effect the presence of a last pointer will have on the time complexities of the operations above.
  • Explain the trade-offs between persistent and mutable lists.

Persistent Linked Lists

  • Be able to write the code to implement any of the above operations for persistent lists.
  • Show how we can use records to implement a linked list roughly equivalent to the lists that come with Clojure.
  • Show how memory is either shared or copied as a result of a list operation.

Mutable Linked Lists

  • Be able to write the code to implement any of the above operations for persistent lists.
  • Show how we can use a deftype to implement a mutable linked list.
  • Show how a change in one list can become visible to another list when sharing sublists.

Doubly Linked Lists

  • Explain why doubly linked lists are best implemented with mutable data.
  • Write the code to delete or insert an element.
  • Write code to reverse a doubly linked list in-place.
  • Explain how sentinels make the doubly linked list code easier to write.

Stacks

  • Know the three stack operations:
    1. Push \({\cal O}(1)\)
    2. Pop \({\cal O}(1)\)
    3. Top \({\cal O}(1)\)
  • Be able to implement the stack operations using vectors.
  • Be able to implement the stack operations using linked lists.

Queues

Know the three queue operations:

  1. Enqueue \({\cal O}(1)\)
  2. Dequeue \({\cal O}(1)\)
  3. Peek \({\cal O}(1)\)

Be able to implement a queue using vectors.

Be able to implement a queue using a single mutable linked list.

Be able to implement a queue using two immutable linked lists.

  • Explain when to flip one list to the other.

  • Explain the concept of amortized complexity.

  • Be able to explain the purpose of a list zipper.
  • Given an implementation of a list zipper, show the following operations:
    • forward
    • backward
    • swap
    • reset
    • get
    • convert to list

Binary Search Trees

  • Be able to implement the three operations:
    1. Add \({\cal O}(\lg n)\)
    2. Find \({\cal O}(\lg n)\)
    3. Delete \({\cal O}(\lg n)\)
  • Explain inorder predecessor and inorder successor. Be able to code them and explain what they are for.
  • Explain why the worst case for BST operations is \({\cal O}(n)\), and how to bring that about.

Tree Traversals

  • Provide the following traversal patterns for a given tree:
    • Preorder
    • Postorder
    • Inorder
    • Level-order
  • Given a preorder traversal, reconstruct the tree.
  • Given a postorder traversal, reconstruct the tree.

Quadratic Sorting

  • Given sample data, show which elements insertion sort and selection sort move first.
  • Explain the advantages that selection sort and insertion sort both have.

\({\cal O}(n \log n)\) Sorting

  • Show how selection sort works with linked lists.
  • Explain why selection sort needs extra data when sorting arrays.
  • Show how to run the partitioning step of quicksort.
  • Explain the situation in which quicksort performs badly, and how to fix it.

Locality

  • Explain temporal locality.
  • Explain spatial locality.
  • Show the result of using the move to front heuristic.
  • Show the result of using the swap heuristic.
  • Explain the tradeoffs involved in using the move to front vs the swap heuristic.

Sequences

  • Explain the concept of a sequence.
  • Explain how sequences are different than lists.
  • Explain what happens when you call next or rest on a vector.

Heaps

  • Know the time complexity for the standard heap operations.
  • Show the effect of an insert and a delete on an array implementation of a heap.
  • Explain the heapify operation and the benefit it brings us.

Hash Tables

  • Given sample data and an array, show how a hash table insert works for the following collision resolution methods:
    • Separate Chaining
    • Linear Probing
    • Quadratic Probing
    • Double Hashing
  • Explain the time complexity in the usual case and when the table becomes full.
  • Explain why we need to mark array entries after an element within them has been deleted.

Up Trees

  • Show how up-tree find and union work.
  • Explain how the path compression operation works.
  • Give some example scenarios where up-trees are a good data structure to use.
